## 🚀 Engine Features

### Main Parameters
- **totalTicks**: Animation duration (20 ticks = 1 second)
- **freezePlayer**: Freeze the player (Slowness, Levitation, Jump Boost)
- **strikeLightningAtEnd**: Lightning at the end of the animation
- **stripPlayer**: Strip the player (remove armor and drop items)
- **itemDropIntervalTicks**: Item drop interval (in ticks)
- **customLevitationAmplifier**: Custom levitation strength (-1 = standard)

### Player Movement (movement)
- **startTick/endTick**: Movement period
- **offsetX/Y/Z**: Coordinate offset
- **smooth**: Smooth movement with interpolation

### Stages
- ✅ **Parallel execution** - multiple stages simultaneously
- ✅ **Time overlap** - Stage 1 (ticks 1-60) + Stage 2 (ticks 20-80)
- ✅ **Independent timelines** - each stage has its own startTick/endTick
- ✅ **On-screen titles** - show text to the player
- ✅ **Lightning during animation** - lightning strikes at specific ticks
- ✅ **Particle trail** - particles behind the player during movement

### Particle Shapes
1. **POINT** - single point
2. **RISING_SPIRAL** - ascending spiral (levitation)
3. **SPHERE** - sphere (Fibonacci algorithm)
4. **EXPLOSION** - explosion (random directions)
5. **HELIX** - double helix (DNA effect)
6. **CIRCLE** - horizontal circle

### Transforms
- **radiusStart/End** - radius change from start to end of stage
- **heightOffsetStart/End** - height change (levitation)
- **Linear interpolation** - smooth transitions

### Colored Particles
- **isColored: true** - enable custom color
- **red/green/blue** - RGB values (0-255)
- **particleSize** - particle size (0.1-3.0)
- Works with **REDSTONE/DUST** particles

### Sounds
- **loop: true** - sound looping
- **loopIntervalTicks** - repeat interval
- **volume/pitch** - volume and pitch

### Titles
- **showAtTick** - tick to show the title relative to stage start
- **title** - title text (supports color codes §)
- **subtitle** - subtitle text
- **fadeIn/stay/fadeOut** - fade-in/display/fade-out time (in ticks)

### Lightning During Animation
- **lightningTicks** - list of ticks for lightning strikes (e.g.: [10, 30, 50])

### Particle Trail
- **particleType** - particle type
- **intervalTicks** - particle spawn interval
- **count** - number of particles per spawn
- **isColored** - use custom color
- **red/green/blue** - RGB values (0-255)
- **particleSize** - particle size
- **offsetX/Y/Z** - particle spread

## 🤖 How to Work with AI

### AI Prompt Structure

```
Create a YAML ban animation configuration for a Minecraft plugin with the following parameters:

THEME: [theme description - fire, ice, magic, space, etc.]
DURATION: [number of seconds]
MOOD: [dramatic, calm, aggressive, mystical]

REQUIREMENTS:
- Use [number] parallel stages
- Main colors: [RGB values or names]
- Shapes: [SPHERE, SPIRAL, EXPLOSION, etc.]
- Sounds: [yes/no, which ones]

STRUCTURE:
totalTicks: [duration * 20]
freezePlayer: true
strikeLightningAtEnd: [true/false]
stages:
  - [stage 1 description]
  - [stage 2 description]
  - ...
```

### AI Keywords

**For shapes:**
- "ascending spiral" → RISING_SPIRAL
- "sphere" / "ball" → SPHERE
- "explosion" → EXPLOSION
- "double helix" / "DNA" → HELIX
- "circle" / "ring" → CIRCLE

**For effects:**
- "fire" → FLAME
- "smoke" → SMOKE_LARGE
- "magic" → SPELL_WITCH
- "portal" → PORTAL
- "lightning" → END_ROD
- "soul" → SOUL

**For transforms:**
- "expanding" → radiusStart < radiusEnd
- "contracting" → radiusStart > radiusEnd
- "rising" → heightOffsetStart < heightOffsetEnd
- "descending" → heightOffsetStart > heightOffsetEnd

## 💡 Tips and Tricks

### New Features

1. **Stripping the Player**
   ```yaml
   stripPlayer: true
   itemDropIntervalTicks: 5  # Drop an item every 5 ticks
   ```
   - All armor is removed immediately
   - Items are dropped gradually with a pickup sound
   - Dropped items are automatically removed after the animation
   - Inventory is cleared to prevent duplication

2. **Custom Levitation**
   ```yaml
   customLevitationAmplifier: 3  # Strong levitation
   # -1 = standard (1)
   # 0 = no levitation
   # 1-10 = varying strength
   ```

3. **Player Movement**
   ```yaml
   movement:
     startTick: 1
     endTick: 60
     offsetX: 0.0
     offsetY: 5.0  # Raise by 5 blocks
     offsetZ: 0.0
     smooth: true  # Smooth movement
   ```

4. **On-screen Titles**
   ```yaml
   titles:
     - showAtTick: 0
       title: "§c§lYOU ARE BANNED"
       subtitle: "§7Reason: cheats"
       fadeIn: 10
       stay: 40
       fadeOut: 10
   ```
   - Supports color codes (§c = red, §l = bold)
   - Multiple titles can be used in one stage

5. **Lightning During Animation**
   ```yaml
   lightningTicks: [10, 30, 50]  # Lightning at ticks 10, 30, and 50 of the stage
   ```

6. **Particle Trail**
   ```yaml
   particleTrail:
     particleType: "REDSTONE"
     intervalTicks: 2
     count: 5
     isColored: true
     red: 255
     green: 0
     blue: 0
     particleSize: 1.5
     offsetX: 0.3
     offsetY: 0.3
     offsetZ: 0.3
   ```

### For Creating Dramatic Animations

1. **Use Buildup**
   ```yaml
   Stage 1: Quiet start (small radius, few particles)
   Stage 2: Buildup (increasing radius and count)
   Stage 3: Climax (maximum effects)
   Stage 4: Explosion (EXPLOSION shape)
   ```

2. **Combine Shapes**
   ```yaml
   - RISING_SPIRAL for levitation
   - SPHERE for surrounding
   - EXPLOSION for the finale
   ```

3. **Use Sounds for Atmosphere**
   ```yaml
   - Looped sounds (loop: true) for background
   - Single sounds for accents
   - Loud sound (volume: 2.0) for the finale
   ```

### For Performance Optimization

1. **Use intervalTicks**
   ```yaml
   intervalTicks: 2  # Spawn every 2 ticks instead of every tick
   ```

2. **Limit count**
   ```yaml
   count: 20  # 20-30 particles is enough for SPHERE
   ```

3. **Don't Overload with Parallel Stages**
   ```yaml
   # Good: 2-3 stages simultaneously
   # Bad: 5+ stages simultaneously
   ```

### For Creating Smooth Transitions

1. **Overlap Stages**
   ```yaml
   Stage 1: startTick: 1,  endTick: 60
   Stage 2: startTick: 40, endTick: 80  # Overlap at ticks 40-60
   ```

2. **Use Interpolation**
   ```yaml
   radiusStart: 3.0
   radiusEnd: 1.0
   # Radius smoothly decreases from 3.0 to 1.0
   ```

3. **Sync with Sounds**
   ```yaml
   # Sound at the moment of stage change
   playAtTick: 0  # Start of the stage
   ```
